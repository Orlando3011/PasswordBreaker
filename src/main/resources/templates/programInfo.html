General information

This program is directed towards novice programmers, to help them with using software design patterns. With the use of simple forms, user can generate source code of a design pattern in a few moments. Each source file can be downloaded as a .java file, and edited remotely in one's machine.

Architecture

This application is based on an MVC pattern, which is an architectural pattern (do not mistake it with design patterns). API created in this pattern is divided into three major modules:

  • Model - representation of an API logic - is this case, it contains definition of a Pattern class, and classes that inherits from it.
  • View - it describes the way, in which model is displayed do the user. This application was intended to work as an Web API, so its model contains HTML templates, JS scripts, and CSS files.
  • Controller - it is a link between user and model / views. It changes views according to the user's actions, and makes operation on model elements. PattersGenerator has a controller for each HTML template. Every one of them has methods to handle user requests.

This pattern makes an application easy to understand and develop, and makes writing code easier. The same approach in writing API with user interface is used in C# ASP library.

Technologies used

This project was created with the use of:

  • Maven - managing plugins and external libraries;
  • Java 1.8;
  • Spring Boot - creating first project files, and making configuration;
  • Spring Web - making web application with MVC pattern;
  • HTML 5.1, CSS, JavaScript;
  • Thymeleaf - connecting HTML templates with Java backend;
